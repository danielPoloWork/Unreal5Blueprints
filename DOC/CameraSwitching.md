# Camera Switching

## Overview
This guide provides a detailed explanation of the Blueprint logic used to switch between first-person and third-person cameras in an Unreal Engine project.

## Viepoint Breakdown
To make the Blueprint work correctly, you need to create two camera viewpoints:

### 1. **Add the First-Person Camera**
- Open the `BP_ThirdPersonCharacter` Blueprint.
- Click on `Viewport`.
- From the left menu, navigate to `Components > Add > Camera`.
- Move the new camera into `Mesh (CharacterMesh0)` and name it `cameraFirstPerson`.
- Set the following properties for `cameraFirstPerson`:
     - **Location**: `0.0, 0.0, -89.0`
     - **Rotation**: `0.0, 0.0, 270.0`
     - **Scale**: `1.0, 1.0, 1.0`
     - Adjust these settings based on your character's proportions if needed.

### 2. **Rename and Configure the Third-Person Camera**
- Rename the existing camera (`CameraBoom root`) to `cameraThirdPerson`.
- Set the following properties for `cameraThirdPerson`:
     - **Location**: `-400.0, 0.0, 0.0`
     - **Rotation**: `0.0, 0.0, 0.0`
     - **Scale**: `1.0, 1.0, 1.0`

## Blueprint Breakdown

### 1. **Input Action Event**
The process starts with an **InputAction Event**:
- **`K2Node_InputAction_0`** listens for the "switchCamera()" action.
- It has two output execution pins:
  - `Pressed`: Executes when the key is pressed.
  - `Released`: Executes when the key is released.

### 2. **Flip-Flop Node**
- **`K2Node_MacroInstance_0`** represents a Flip-Flop macro from Unreal Engine's standard macros.
- This node toggles between two execution paths (`A` and `B`) each time it is triggered.
- The `Pressed` pin from `K2Node_InputAction_0` connects to the Flip-Flop node.

### 3. **Camera Switching Logic**

#### Path A (Switch to First-Person View)
1. `K2Node_VariableGet_19` retrieves the **cameraFirstPerson** variable.
2. `K2Node_CallFunction_20` calls the `SetActive(true)` function to enable the first-person camera.
3. `K2Node_CallFunction_22` disables the third-person camera.
4. `K2Node_VariableSet_7` enables **bUseControllerRotationYaw**, aligning the character rotation with the camera.

#### Path B (Switch to Third-Person View)
1. `K2Node_VariableGet_20` retrieves the **cameraThirdPerson** variable.
2. `K2Node_CallFunction_21` calls the `SetActive(true)` function to enable the third-person camera.
3. `K2Node_CallFunction_23` disables the first-person camera.
4. `K2Node_VariableSet_5` disables **bUseControllerRotationYaw** to allow free camera movement.
